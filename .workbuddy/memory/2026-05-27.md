## 修复浏览器通知不生效的问题

**问题**：番茄倒计时完成后，用户没有收到浏览器通知。

**根因分析**：
- `notify()` 返回 void，失败时完全静默无日志
- `requestPermission()` 未 await，结果被忽略
- macOS Chrome 前台标签页经常不弹通知横幅（浏览器行为，非 bug）
- 没有应用内降级通知方案

**修复**：
1. `hooks.ts` — `notify` 改为返回 `Promise<boolean>`，增加每条代码路径的详细日志
2. `hooks.ts` — 添加 `requireInteraction: true`，防止 macOS 自动关闭通知
3. `hooks.ts` — SW 路径和直接 Notification 路径分别 try/catch
4. 新建 `Toast.tsx` — 应用内通知组件，带入场/退场动画，5秒自动消失
5. `page.tsx` — `handleFocusComplete` 中始终显示 toast 作为兜底通知

## 修复部署后浏览器通知不弹（本地正常）

**问题**：本地 localhost 通知正常，部署到服务器后无通知。

**根因**：
1. Service Worker 路径写死 `/sw.js`，但项目配置了 `basePath: "/tomato-clock"`（`output: "export"`），部署后实际路径是 `/tomato-clock/sw.js`，SW 注册 404
2. Notification API 要求安全上下文（HTTPS 或 localhost），部署后走 HTTP 时所有通知 API 静默失败

**修复**（hooks.ts）：
- `getSwPath()` 动态获取 SW 路径：从 `<base>` 标签或 `window.location` 推断，兼容 basePath
- `requestPermission` 增加 `window.isSecureContext` 检查，非安全上下文时返回 false + 设置 `secureError`
- `notify` 同样增加 `isSecureContext` 检查
- 新增 `secureError` 状态，可给 UI 层展示错误原因
- 需要 `useState` import

**服务器侧需求**：部署必须走 HTTPS，否则 Notification API 无法使用
- 更新了 page.tsx 中的 `useNotification()` 解构，但 page.tsx 当前未使用 `secureError`，可后续在 UI 中展示友好提示

## 修复 /calendar/ tab 高亮失效

- **问题**: 生产环境 `next.config.ts` 配置了 `trailingSlash: true`，`usePathname()` 返回 `/calendar/`（带尾部斜杠），而 `AppNav.tsx` 的 `links` 中 `href` 为 `/calendar`（无斜杠），精确匹配 `===` 失败
- **修复**: `app/components/AppNav.tsx` 中 normalize pathname 和 href，统一去掉尾部斜杠后再比较
